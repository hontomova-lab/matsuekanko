# スプレッドシート ⇄ ガントチャート 接続設計

作成日：2026年6月12日
対象：`task_gantt.html`（ガントチャート）／ Googleスプレッドシート「松江タスク管理」

---

## 1. 接続方式（全体像）

```
[Googleスプレッドシート]  ──ウェブに公開(CSV)──▶  [CSV公開URL]
                                                      │ fetch
                                                      ▼
                                      [task_gantt.html（GitHub Pages）]
                                          画面のURL欄に貼り付け→保存→自動描画
```

* スプレッドシートを **「ウェブに公開（CSV形式）」** し、出力されるCSV URLをガントチャートが `fetch` で読み込む方式。
* CORSの制約に強く、参考にしたプロポーザル管理アプリと同じ実績のある方式。
* ガントチャート側はURLを**ブラウザに保存（localStorage）**するため、一度設定すれば次回以降は自動で最新データを読み込む。

---

## 2. 事前準備（重要）

現在のファイルは **Excel（.xlsx）をアップロードした状態**（URLに `rtpof=true`）。
このままでは「ウェブに公開」が使えないため、**ネイティブのGoogleスプレッドシートに変換**する。

1. 対象ファイルを開く
2. メニュー **ファイル → Google スプレッドシートとして保存**
3. 変換後の新しいファイル（こちらが本番用）を以降で使用する

---

## 3. 設定手順

### STEP 1：シート構成の確認
* 1行目：タイトル（`松江市エリアブランディング戦略策定 タスク管理`）→ **そのままでOK**
* 2行目：見出し（大項目／中項目／タスク名／…）
* 3行目以降：データ

> ガントチャート側は「タスク名」を含む行を見出しとして**自動検出**するため、1行目のタイトル行があっても問題なく読み込めるよう改修済み。

### STEP 2：共有設定
* 右上 **共有** → 「一般的なアクセス」を **「リンクを知っている全員」＝閲覧者** に設定。
  （※発注者・各担当者の編集は、別途メールアドレス指定で「編集者」権限を付与）

### STEP 3：ウェブに公開（CSV）
1. メニュー **ファイル → 共有 → ウェブに公開**
2. 「リンク」タブ → 対象を **シート「タスク一覧」** に指定
3. 形式を **カンマ区切り値（.csv）** に変更
4. **公開** をクリックし、表示されたURLをコピー
   * URLの形式例：
     `https://docs.google.com/spreadsheets/d/e/2PACX-xxxx/pub?gid=0&single=true&output=csv`

### STEP 4：ガントチャートに接続
1. `task_gantt.html` をブラウザで開く
2. 画面上部の **「スプレッドシートCSV URL」欄** に STEP 3 のURLを貼り付け
3. **「URLを保存して読込」** をクリック → ガントチャートが描画される
   * 以降はURLがブラウザに保存され、開くたびに自動で最新データを取得
   * 別のPC・GitHub Pages公開版でも、同じ操作でURLを設定すればよい

---

## 4. データ更新の流れ（運用）

1. 関係者がスプレッドシートを編集（開始日・終了日・打合せ・提出・承認・ステータス等を入力）
2. ガントチャート画面の **「↺ 更新」** ボタンを押す → 最新状態を再取得
   * ※「ウェブに公開」のCSVはGoogle側で数分キャッシュされる場合があり、反映に最大5分程度かかることがある

---

## 5. ガントチャートへの反映ルール（列の対応）

| スプレッドシート列 | ガントチャートでの表現 |
| :--- | :--- |
| 大項目 | グループ見出し（区切り行） |
| 中項目 | 左パネルのサブテキスト |
| タスク名 | 左パネルのメインテキスト |
| 担当組織 | 担当バッジ（組織ごとに色分け） |
| 担当者名 | バッジ横に表示 |
| ステータス | バーの色／「完了を非表示」の対象判定 |
| 開始日・終了日 | スケジュールバー（帯）の始点〜終点 |
| 打合せ | ● 青アイコン |
| 提出 | ◆ 赤アイコン |
| 承認 | ◎ 緑（二重丸）アイコン |
| 備考 | ツールチップに表示 |

* 日付は `2026/6/15` のような形式で入力（`2026/6` のように日を省くと「6月頃」として扱う）。
* 「開始日」と「終了日」が両方入っている行のみバーを描画。片方のみの場合は点表示。

---

## 6. GitHub Pages 公開時の注意

* 公開リポジトリ：`https://github.com/hontomova-lab/matsuekanko`
* `task_gantt.html` をリポジトリに配置（`index.html` にリネームするとトップで表示される）。
* GitHub Pages（https配信）からはCORS制約なくCSVを読み込めるため、URL欄に設定したCSVがそのまま動作する。
* `file://`（ローカルでHTMLをダブルクリック）で開くと外部URL読込はCORSで失敗する。その場合は **「📂 CSV読込」** で手元のCSVを読み込んで確認する。

---

## 7. トラブル時の確認ポイント

| 症状 | 確認 |
| :--- | :--- |
| 「データがありません」 | URL欄が空 → STEP 4 を実施。または共有が「全員閲覧」になっているか |
| 「HTTP 404 / 401」エラー | 公開設定（ウェブに公開）が未実施、または共有権限不足 |
| 列がずれる／空になる | 見出し行に「タスク名」があるか（自動検出のキー） |
| 最新の編集が反映されない | Google側キャッシュ。数分待って「↺ 更新」 |
