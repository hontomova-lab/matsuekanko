# 松江市エリアブランディング戦略策定 タスク管理ガントチャート 要件定義書

作成日：2026年6月12日

## 1. システム構成・運用方針

* **データソース:** Googleスプレッドシート（タスク一覧・スケジュールを入力）
* **表示アプリ:** HTML/JSによるシングルページWebアプリ（GitHub Pagesでホスティング）
* **ベースコード:** 既存の「プロポーザル スケジュール管理 v3（`index_v3.html`）」を流用・カスタマイズ
* **利用対象者:** 発注者（松江観光協会）、受注者コンソーシアム（エブリプラン、グレイスデザイン）の各担当者
* **権限:** 全関係者がスプレッドシートの閲覧・編集、およびWebアプリの閲覧可能

## 2. スプレッドシート列構成（データ構造）

プロポーザル管理用の構成から、単一プロジェクトのタスク管理に最適化した列構成に変更する。

| 列名 | 説明 | ガントチャートでの表示・用途 |
| :--- | :--- | :--- |
| **大項目** | 「1. 現状分析」などのフェーズ | 行のグループ分け（セパレーター）として利用 |
| **中項目** | 大項目に紐づく詳細フェーズ | ツールチップ等に表示（必要に応じて） |
| **タスク名** | 「既存資料の確認」など | 左側のメインテキスト |
| **担当組織** | エブリプラン / グレイスデザイン / 両者 / 観光協会 | 担当バッジとして表示（組織ごとに色分け） |
| **担当者名** | 実際の作業者名 | 担当バッジの横、またはツールチップに表示 |
| **ステータス** | 未着手 / 進行中 / 完了 など | バーの色分け、および「完了を非表示」の判定 |
| **開始日** | タスクの開始予定日 | バーの始点（YYYY/M/D形式） |
| **終了日** | タスクの終了予定日 | バーの終点（YYYY/M/D形式） |
| **打合せ** | キックオフや定例会議の日程 | アイコン（例：● 青色） |
| **提出** | 成果物や資料の提出日 | アイコン（例：◆ 赤色） |
| **承認** | 発注者（観光協会）の承認日 | アイコン（例：◎ 緑色二重丸） |
| **備考** | メモ | ツールチップ内に表示 |

## 3. ガントチャート（Webアプリ）の変更点

既存の `index_v3.html` から以下の改修を行う。

### 3.1. 画面レイアウト・表示内容
* **タイトル・ヘッダー:** 「松江市エリアブランディング戦略策定 タスク管理」に変更。
* **左側情報パネル:** 
  * メインテキスト：タスク名
  * サブテキスト：中項目など
  * 担当表示：「担当組織」で色分けしたバッジを表示し、その横に「担当者名」を併記。
* **グループ分け:** 「部署」での区切り線ロジックを、「大項目」での区切り線に変更。

### 3.2. スケジュールバーの描画ロジック
* 複数のマイルストーンから最小・最大日を算出する方式を廃止し、**「開始日」から「終了日」までの期間で帯（バー）を描画**する方式に変更。

### 3.3. マイルストーン（アイコン）の再定義
プロポーザル用のアイコン設定を破棄し、タスク管理用の3種類に再定義する。

| 項目名 | 形状 | 色（案） |
| :--- | :--- | :--- |
| 打合せ | `circle` (● 塗り円) | 青系 |
| 提出 | `diamond` (◆ 塗り菱形) | 赤系 |
| 承認 | `doubleCircle` (◎ 二重丸) | 緑系 |

### 3.4. コントロールバー・フィルタ機能
* **並び替え:** 「大項目順」「開始日順」「ステータス順」などに選択肢を調整。
* **完了を非表示:** ステータス列が「完了」となっているタスクを非表示にするトグルボタンとして機能させる。

## 4. 開発手順

1. 本要件定義書の確定

5. 本番用スプレッドシートの作成とURL連携テスト
