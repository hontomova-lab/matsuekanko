## 1. システムの概要と各種URL

スプレッドシートで入力したタスク一覧を、見やすいガントチャート形式でWeb上に自動描画するシステムです。

* **ガントチャート（閲覧用Webアプリ）**
  * URL: `https://hontomova-lab.github.io/matsuekanko/`
  * ホスティング: GitHub Pages (`hontomova-lab/matsuekanko` リポジトリの `main` ブランチ)
* **データソース（入力用スプレッドシート）**

  * 権限: リンクを知っている全員が閲覧可能（編集権限は各担当者に個別付与）

---

## 2. システム構成と連携の仕組み

1. **Googleスプレッドシート**の「タスク一覧」シートを **「ウェブに公開（CSV形式）」** しています。
2. **GitHub Pages** 上のWebアプリ（`index.html`）が、ページ読み込み時に上記の公開CSV URLへアクセス（`fetch`）し、最新データを取得してガントチャートを描画します。
3. スプレッドシートを更新した場合、ガントチャート画面で「↺ 更新」ボタンを押すか、ページをリロードするだけで最新状態が反映されます（※Google側のキャッシュにより、反映に最大5分程度かかる場合があります）。

## 3. スプレッドシートの運用ルール（データ構造）

スプレッドシートの2行目が見出し行となっており、以下の列構成で運用します。
※ガントチャートは「タスク名」という文字列を含む行を見出しとして自動検出するため、1行目のタイトル行はそのまま残して問題ありません。

| 列名 | 入力内容・用途 | ガントチャートでの表示 |
| :--- | :--- | :--- |
| **大項目** | フェーズ（例：1. 現状調査WS） | 行のグループ分け（セパレーター） |
| **中項目** | 詳細グループ（例：1-1 チラシ） | 左パネルのサブテキスト・ツールチップ |
| **タスク名** | 個別の作業名 | 左パネルのメインテキスト |
| **担当組織** | エブリプラン / グレイスデザイン / 両者 / 観光協会 | 担当バッジ（組織ごとに色分け） |
| **担当者名** | 実際の作業者名 | 担当バッジの横に表示 |
| **ステータス** | 未着手 / 進行中 / 完了 / 保留 | バーの色分け。「完了」は非表示フィルタの対象 |
| **開始日** | タスクの開始予定日（yyyy/m/d） | スケジュールバー（帯）の始点 |
| **終了日** | タスクの終了予定日（yyyy/m/d） | スケジュールバー（帯）の終点 |
| **打合せ** | 会議等の日程（yyyy/m/d） | ● 青色アイコン |
| **提出** | 成果物の提出日（yyyy/m/d） | ◆ 赤色アイコン |
| **承認** | 発注者の承認日（yyyy/m/d） | ◎ 緑色二重丸アイコン |
| **備考** | メモ | ツールチップ内に表示 |

## 4. ガントチャート画面の主な機能

* **完了を非表示トグル:** ステータスが「完了」のタスクをワンクリックで隠し、進行中のタスクに集中できます。
* **並び替え:** 「大項目 → 開始日」「ステータス → 開始日」「開始日のみ」の3パターンでソート可能です。
* **ツールチップ:** ガントチャートのバーやアイコンにマウスカーソルを合わせると、タスクの詳細情報がポップアップ表示されます。
* **ローカルCSV読込:** ネットワーク制限等でスプレッドシートと連携できない場合、手元にダウンロードしたCSVファイルを直接読み込ませて表示することが可能です。

## 5. 今後のメンテナンス・修正方法

### 5.1. スプレッドシートを変更した場合（ファイルを作り直した場合）
万が一、スプレッドシートを新しく作り直した場合は、CSVの公開URLが変わります。
1. 新しいスプレッドシートで「ウェブに公開（CSV）」を実施し、URLを取得します。
2. `index.html`（または `task_gantt.html`）のソースコード内にある以下の部分を書き換えて、GitHubへプッシュしてください。
   ```javascript
   const SHEET_CSV_URL = '新しいCSV公開URL';
   ```

### 5.2. コードの修正（GitHubへの反映）
ローカルの `c:\Users\takeshita\Documents\松江観光\タスク管理` フォルダがGitリポジトリになっています。
HTMLファイル等を修正した場合は、以下のコマンドでGitHubへ反映（デプロイ）してください。
```bash
git add .
git commit -m "修正内容のメモ"
git push origin main
```
プッシュ後、数分で GitHub Pages に自動反映されます。
