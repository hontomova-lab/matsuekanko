# プロポーザル管理ガントチャート｜要件定義 v3

作成：守山　2026年6月2日
前バージョン：`プロポーザル管理_ガントチャート_要件定義_v2.md`（2026年4月30日）

---

## v3 で追加・変更する要件（要約）

1. **「質問締切」と「参加表明」のマーク分離**
   v2 では両方とも青い ●（塗り円）で見分けがつかなかった。参加表明を ○（中空円）に変更し、同系の青のまま塗りの有無で区別する。
2. **「完了を非表示」フィルタボタンの追加**
   完了案件（受注＋不受注）をワンクリックで隠せるトグルボタンをコントロールバーに追加。進行中案件に集中したいときに視界をすっきりさせる。

いずれもデータ構造（スプレッドシート列構成）は変更しない。表示ロジックのみの改修で、v2 と完全後方互換。

---

## 1. 質問締切／参加表明のマーク分離

### 背景
v2 ではマイルストーン定義で「質問締切」「参加表明」がともに `shape: 'circle'` / `color: '#2196F3'`（塗りの青円）。同一行に両方が近接して並ぶと判別できなかった。

### 変更内容

| 列名 | v2 | v3 | 形状 | 色 |
|---|---|---|---|---|
| 質問締切 | ● 塗り円 | ● 塗り円（**変更なし**） | `circle` | 青 `#2196F3` |
| 参加表明 | ● 塗り円 | **○ 中空円** | `circleOutline`（新規） | 青 `#2196F3`（枠線） |

- 色は両方とも青系で統一し、「公募〜提出前の事前手続き」というグルーピング感を保持
- 塗り（●）＝確認のみで済む質問締切／中空（○）＝意思表示が必要な参加表明、という直感に沿わせる
- 実装：`drawRow()` のマイルストーン描画に `circleOutline` 分岐を追加（`fill="#fff"` + `stroke` の中空円）

### 凡例バーの変更
v2 の `● 質問締切・参加表明`（1項目）を、以下の2項目に分割：
- `● 質問締切`（青）
- `○ 参加表明`（青）

## 2. 「完了を非表示」フィルタボタン

### 背景
案件が増えると完了済み（受注・不受注）が一覧に積み上がり、進行中案件が埋もれる。完了案件をワンクリックで隠せるようにする。

### 仕様

| 項目 | 内容 |
|---|---|
| 設置場所 | コントロールバー（並び替えプルダウンの右、更新ボタンの左） |
| 初期状態 | **全件表示**（完了案件も表示）。データの取りこぼし感を避けるため |
| 操作 | ボタンを押すたびに「非表示 ⇄ 表示」をトグル |
| 対象フェーズ | `完了（受注）` と `完了（不受注）` の2フェーズ |
| ボタン表記 | 全件表示中：「完了を非表示」（青 `#5C6BC0`）／非表示中：「完了を表示」（緑 `#43A047`） |
| 件数表示 | 非表示中はステータス欄に「表示 N 件（完了 M 件を非表示 / 全 X 件）」を表示 |
| 全件が完了で空になる場合 | 「表示できる案件がありません（すべて完了案件のため非表示中）」を空状態表示 |

### 挙動の詳細
- フィルタは `render()` 冒頭で作業用配列 `visibleRows` を生成して適用。**元データ `rows` は破壊しない**ため、再表示で完全復帰する
- 日付範囲の自動計算・並び替え・部署セパレーターはすべて `visibleRows` ベースで再計算される（隠した案件は範囲算出にも影響しない）
- 並び替えやCSV再読込をしてもフィルタ状態は維持される

## 3. スプレッドシート列構成（v3）

**v2 から変更なし。** 既存スプレッドシートをそのまま利用できる。

```
A: No.
B: 企画番号
C: 案件名
D: 発注者
E: 金額（税込）
F: 部署
G: フェーズ           ← 完了（受注）/完了（不受注）がフィルタ対象
H: 次回問い合わせ予定
I: 予算化見込み
J: 公募開始
K: 質問締切           ← ● 塗り円
L: 参加表明           ← ○ 中空円（v3変更）
M: 企画提出
N: プレゼン
O: 主担当
```

## 4. UIへの影響

### コントロールバー（追加要素）
- **「完了を非表示」トグルボタン**（並び替えの右）

### 凡例バー（変更）
- `● 質問締切・参加表明`（1項目）→ `● 質問締切` ＋ `○ 参加表明`（2項目）に分割

---

## 5. 影響範囲・互換性

- スプレッドシート構成は無変更。v2 データをそのまま表示可能
- 参加表明の形状変更は表示のみ。データ・ツールチップ・並び替えロジックに影響なし
- 完了フィルタは非破壊（`rows` を保持し `visibleRows` で描画）。トグルで完全復帰
- v1・v2 の HTML はそのまま残置（`index.html` / `index_v2.html`）

---

## 6. 拡張候補（v3 でも対象外）

- 部署フィルター（チェックボックスで絞り込み）
- フェーズ別フィルター（完了以外も任意に絞る一般化）
- 担当者フィルター／分野タグ／受注確率・優先度
- 「次回問い合わせ予定」が過ぎたらアラート表示
- 表示期間プリセット／印刷用レイアウト（A3横）
- フィルタ状態のURLパラメータ保持（共有時に状態を引き継ぐ）

---

## 開発手順

1. 本要件定義（v3）レビュー・確定
2. `index_v2.html` を複製して `index_v3.html` を作成
3. 以下を実装：
   - MILESTONES「参加表明」を `shape: 'circleOutline'` に変更
   - `drawRow()` に `circleOutline` 描画分岐を追加
   - コントロールバーに「完了を非表示」トグルボタンを追加
   - `render()` に `visibleRows` フィルタ・件数表示・空状態メッセージを実装
   - 凡例バーの質問締切／参加表明を2分割
   - ヘッダーのバージョン表記を v3 に更新
4. サンプルデータ（完了案件2件を含む）で動作確認
5. 本番スプレッドシートに切り替えて検証
6. GitHub Pages の配信元を `index_v3.html` に切替（または `index.html` として配置）
7. サイボウズで案内文（v3 アップデート）
